How to use this extension:

  1. Get a ReCAPTCHA key here
  2. In the template where you want to use ReCAPTCHA add
        {captcha publickey="<public-key>"}
    in the form
  3. In the controller that needs to check add
        $r = captcha::checkCaptcha(<private-key>);
  4. If $r->isValid is true, the ReCAPTCHA was answered correctly, else the error is provided in $r->error

Errors

  • invalid-site-private-key
    Invalid private key used for this site, check if the key is for the domain it is used on.
    Possible solution:
    • Did you swap the public and private key?
    • Did you make sure to copy the entire key, with all hyphens and underscores but without any spaces? The key should be exactly 40 characters long.
  • invalid-request-cookie
    The challenge parameter of the verify script was incorrect 
  • incorrect-captcha-sol
    The ReCAPTCHA solution was incorrect
  • recaptcha-not-reachable
    The ReCAPTCHA verification server was not reachable 

Themes

A few themes are supported by this extension. They can be set by adding the "theme" parameter to the {captcha} call.

An example looks as follows:

{captcha publickey="<publickey>" theme="white"}

Possible themes are: red, white, blackglass or clean. See customizing for theme examples.

Languages

Like themes, different languages can be used for ReCAPTCHA. When no language has been set the extension will look which language is set for the page and if it's a known language it will use that language.

You can force the language by adding the "lang" parameter to the {captcha} call.

An example looks as follows:

{captcha publickey="<public-key>" lang="de"}

Possible languages are: en, nl, fr, de, pt, ru, es, tr.

Private and public key

The keys can be set hard-coded in the template and controller, but it is better to use the page config to handle these values.

An example for plugin config is:

"plugins": {

"form": [

"Form",

{

"useCaptcha": {

"label": "Use captcha?",

"type": "checkbox",

"depends": {

"pubkey": {

"needValue": 1,

"label": "Public key",

"type": "text"

},

"privkey": {

"needValue": 1,

"label": "Private key",

"type": "text"

}

}

}

}

]

}

As you can see, the keys are provided in the plugin config now instead of hardcoded in the template and controller. This makes your extension more flexible and easier to use.
