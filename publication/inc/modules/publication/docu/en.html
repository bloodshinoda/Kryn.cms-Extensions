Setup

This extension provides two plugins. You have to place both in seperated pages to run this extensions correctly.

News List

O Display a list of news by one or more categories.

 

URL-Scheme

 

For example if current page, which the list-plugin is placed, is: http://mydomain.de/news/

 

  • http://mydomain.de/news/ => page 1
  • http://mydomain.de/news/1/ => page 1
  • http://mydomain.de/news/2/ => page 2
    http://mydomain.de/news/3/ => page 3
  • http://mydomain.de/news/n/ => page n

 

News detail

This plugin read the first extra argument in the url and display the detail news.

 

URL-Scheme


For example if current page, which the detail-plugin is placed, is: http://mydomain.de/news-detail/

 

  • http://mydomain.de/news-detail/my-news-title-woho/3/
    => display news item with id=3 and title 'my-news-title-woho'
  • http://mydomain.de/news-detail/my-news-title-2-woho/4/
    => display news item with id=4 and title 'my-news-title-2-woho'

Template vars

news-list

(inc/template/publication/news/list/)

  • array $items
    • each news entry as array $item
      • string title
        • json files
        • You can use this var with json_decode:
          {foreach from=$item.files|json_decode item=file}
              {$file}
          {/foreach}
      • string intro
      • string content
      • timestamp releaseAt
      • timestamp releaseDate
      • int (0/1) deactivate
        • string introImage
          contains the path of the image (example: myImages/image.jpg)
        • string introImage2
        • same as introImage
  • int $pages
    contains the count of max pages
  • int $currentNewsPage
    current page

news-detail

(inc/template/publication/news/detail/)

  • array $item
    • Contain same properties as news-list $items
